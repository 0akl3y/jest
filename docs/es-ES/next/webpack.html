Using with webpack

Jest puede ser usado en proyectos que usan webpackpara manejar bienes, estilos y compilación. webpack ofrece algunos desafíos únicos sobre otras herramientas porque integra directamente con su aplicación para permitirle manejar plantillas de estilo, valores como imágenes y fuentes, junto al ecosistema expansivo de lenguaje y herramientas de compilar-a- Javascript.

Un ejemplo de webpack

Comencemos con un tipo común de archivo de configuración webpack y convirtámoslo en una configuración Jest.

// webpack.config.js
module.exports = {
  module: {
    loaders: [
      {exclude: ['node_modules'], loader: 'babel', test: /\.jsx?$/},
      {loader: 'style-loader!css-loader', test: /\.css$/},
      {loader: 'url-loader', test: /\.gif$/},
      {loader: 'file-loader', test: /\.(ttf|eot|svg)$/},
    ],
  },
  resolve: {
    alias: {
      config$: './configs/app-config.js',
      react: './vendor/react-master',
    },
    extensions: ['', 'js', 'jsx'],
    modules: [
      'node_modules',
      'bower_components',
      'shared',
      '/shared/vendor/modules',
    ],
  },
};

Si posee archivos JavaScript que son transformados por Babel, puede permitir funcionamiento para Babel instalando el plug-in babel-jest. Transformaciones de JavaScript no compatibles con Babel pueden ser manejadas con la opción de configuración transform de Jest.

Manejo de activos estáticos

Acto seguido, configuremos Jest para que maneje con elegancia archivos de valores como plantillas de estilo e imágenes. Usualmente, estos archivos no son particularmente útiles en pruebas con lo cual podemos duplicarlos de manera segura. De todas maneras, si está usando Módulos CSS entonces es mejor duplicar un proxy por sus búsquedas ClassName.

// package.json
{
  "jest": {
    "moduleNameMapper": {
      "\\.(jpg|jpeg|png|gif|eot|otf|webp|svg|ttf|woff|woff2|mp4|webm|wav|mp3|m4a|aac|oga)$":
        "<rootDir>/__mocks__/fileMock.js",
      "\\.(css|less)$": "<rootDir>/__mocks__/styleMock.js"
    }
  }
}

Y mockear los archivos propios:

// __mocks__/styleMock.js

module.exports = {};

// __mocks__/fileMock.js

module.exports = 'test-file-stub';

Mockear Modulos de CSS

You can use an ES6 Proxy to mock CSS Modules:

yarn add --dev identity-obj-proxy

Then all your className lookups on the styles object will be returned as-is (e.g., styles.foobar === 'foobar'). Esto es muy útil para las Pruebas de Snapshot de React.

// package.json (for CSS Modules)
{
  "jest": {
    "moduleNameMapper": {
      "\\.(jpg|jpeg|png|gif|eot|otf|webp|svg|ttf|woff|woff2|mp4|webm|wav|mp3|m4a|aac|oga)$":
        "<rootDir>/__mocks__/fileMock.js",
      "\\.(css|less)$": "identity-obj-proxy"
    }
  }
}

Nótese que el Proxy no está encendido en Nodo 6 por defecto. Si aún no está en Nodo 6, asegúrese de invocar a Jest utilizando node --harmony_proxies node_modules/.bin/jest.

If moduleNameMapper cannot fulfill your requirements, you can use Jest's transform config option to specify how assets are transformed. Por ejemplo, un transformador que devuelve el nombre base de un archivo (como require('logo.jpg'); returns 'logo') puede ser escrito como:

// fileTransformer.js
const path = require('path');

module.exports = {
  process(src, filename, config, options) {
    return 'module.exports = ' + JSON.stringify(path.basename(filename)) + ';';
  },
};

// package.json (for custom transformers and CSS Modules)
{
  "jest": {
    "moduleNameMapper": {
      "\\.(css|less)$": "identity-obj-proxy"
    },
    "transform": {
      "\\.(jpg|jpeg|png|gif|eot|otf|webp|svg|ttf|woff|woff2|mp4|webm|wav|mp3|m4a|aac|oga)$":
        "<rootDir>/fileTransformer.js"
    }
  }
}

We've told Jest to ignore files matching a stylesheet or image extension, and instead, require our mock files. Usted puede ajustar la expresión regular para emparejar los tipos de archivo que su configuración de webpack maneja.

Nota: si está usando babel-jest con procesadores de código adicionales, tiene que definir babel-jest explícitamente como un transformador para el código de su Javascript para mapear los archivos .js al módulo babel-jest.

"transform": {
  "^.+\\.js$": "babel-jest",
  "^.+\\.css$": "custom-transformer",
  ...
}

Configurar Jest para que encuentre nuestros archivos

Ahora que Jest sabe cómo procesar nuestros archivos, necesitamos decirle cómo encontrarlos. For webpack's modulesDirectories, and extensions options there are direct analogs in Jest's moduleDirectories and moduleFileExtensions options.

// package.json
{
  "jest": {
    "moduleFileExtensions": ["js", "jsx"],
    "moduleDirectories": ["node_modules", "bower_components", "shared"],

    "moduleNameMapper": {
      "\\.(css|less)$": "<rootDir>/__mocks__/styleMock.js",
      "\\.(gif|ttf|eot|svg)$": "<rootDir>/__mocks__/fileMock.js"
    }
  }
}

Note: <rootDir> is a special token that gets replaced by Jest with the root of your project. Most of the time this will be the folder where your package.json is located unless you specify a custom rootDir option in your configuration.

Similarly webpack's resolve.root option functions like setting the NODE_PATH env variable, which you can set, or make use of the modulePaths option.

// package.json
{
  "jest": {
    "modulePaths": ["/shared/vendor/modules"],
    "moduleFileExtensions": ["js", "jsx"],
    "moduleDirectories": ["node_modules", "bower_components", "shared"],
    "moduleNameMapper": {
      "\\.(css|less)$": "<rootDir>/__mocks__/styleMock.js",
      "\\.(gif|ttf|eot|svg)$": "<rootDir>/__mocks__/fileMock.js"
    }
  }
}

Y finalmente nos queda encargarnos del alias de webpack. Para eso podemos usar otra vez la opción moduleNameMapper.

// package.json
{
  "jest": {
    "modulePaths": ["/shared/vendor/modules"],
    "moduleFileExtensions": ["js", "jsx"],
    "moduleDirectories": ["node_modules", "bower_components", "shared"],

    "moduleNameMapper": {
      "^react(.*)$": "<rootDir>/vendor/react-master$1",
      "^config$": "<rootDir>/configs/app-config.js",

      "\\.(css|less)$": "<rootDir>/__mocks__/styleMock.js",
      "\\.(gif|ttf|eot|svg)$": "<rootDir>/__mocks__/fileMock.js"
    }
  }
}

¡Eso es todo! webpack es una herramienta compleja y flexible, así que quizás tenga que hacer algunos ajustes para manejar las necesidades específicas de su aplicación13 webpack es una herramienta compleja y flexible, así que quizás tenga que hacer algunos ajustes para manejar las necesidades específicas de su aplicación. Afortunadamente para la mayoría de los proyectos, Jest debería ser más que flexible para manejar la configuración de su webpack.

Nota: Para configuraciones más complejas de webpack, puede investigar también acerca de proyectos como.babel-plugin-webpack-loaders.

Utilizando webpack 2

webPack 2 ofrece soporte nativo para los módulos de ES. Sin embargo, Jest funciona en Node y por lo tanto requiere que los módulos ES sean transpilados a los módulos de CommonJS. Como tal, si usas webpack 2, probablemente querrás configurar que Babel transpile módulos de ES a módulos de CommonJS sólo en el entorno de pruebas.

// .babelrc
{
  "presets": [["es2015", {"modules": false}]],

  "env": {
    "test": {
      "plugins": ["transform-es2015-modules-commonjs"]
    }
  }
}

Nota: Jest almacena archivos para acelerar la ejecución de pruebas. Si usted actualizó .babelrc y Jest todavía no funciona, intente ejecutar Jest con --no-cache.

Si utilizas las importaciones dinámicas (import('some-file.js').then (módulo = >...)), es necesario activar el plugin de dynamic-import-node.

// .babelrc
{
  "presets": [["es2015", {"modules": false}]],

  "plugins": ["syntax-dynamic-import"],

  "env": {
    "test": {
      "plugins": ["dynamic-import-node"]
    }
  }
}
