Getting Started

Install Jest using yarn:

yarn add --dev jest

Or npm:

npm install --save-dev jest

Comencemos escribiendo una prueba para una función hipotética que agrega dos números. Primero, crea un archivo sum.js:

function suma(a, b) {
  return a + b;
}
module.exports = suma;

Luego, crea un archivo llamado suma.test.js. Este contendrá nuestra prueba actual:

const suma = require('./suma');

test('sumar 1 + 2 es igual a 3', () => {
  expect(suma(1, 2)).toBe(3);
});

Agrega la siguiente sección a tu package.json:

{
  "scripts": {
    "test": "jest"
  }
}

Finally, run yarn test and Jest will print this message:

PASS  ./suma.test.js
✓ sumar 1 + 2 es igual a 3 (5ms)

Acabas de escribir tu primera prueba usando Jest!

Esta prueba uso expect y toBe para probar que dos valores eran exactamente idénticos. Para aprender sobre las otras cosas que Jest puede probar, vea Using Matchers.

Ejecutando desde la línea de comandos

You can run Jest directly from the CLI (if it's globally available in your PATH, e.g. by yarn global add jest) with variety of useful options.

Here's how to run Jest on files matching my-test, using config.json as a configuration file and display a native OS notification after the run:

jest my-test --notify --config=config.json

Si desea obtener mas información sobre la ejecución a través de la linea de comando jest echa una mirada a la pagina de Jest CLI Options.

Configuración Adicional

Usando Babel

Para usar Babel, instalar el babel-jest y los paquetes regenerator-runtime:

yarn add --dev babel-jest babel-core regenerator-runtime

Observa: Si está utilizando una versión babel 7, entonces tiene que instalar babel-jest con el siguiente comando:

yarn add --dev babel-jest 'babel-core@^7.0.0-0' @babel/core regenerator-runtime


*Note: Explicitly installing `regenerator-runtime` is not needed if you use `npm` 3 or 4 or Yarn*

No olvides agregar el archivo [`.babelrc`](https://babeljs.io/docs/usage/babelrc/) en la carpeta raíz de su proyecto. For example, if you are using ES6 and [React.js](https://facebook.github.io/react/) with the [`babel-preset-env`](https://babeljs.io/docs/plugins/preset-env/) and [`babel-preset-react`](https://babeljs.io/docs/plugins/preset-react/) presets:

```json
{
  "presets": ["env", "react"]
}

Ahora estas preparado para usar todas las características de ES6 y la sintaxis específica de React.

Note: If you are using a more complicated Babel configuration, using Babel's env option, keep in mind that Jest will automatically define NODE_ENV as test. It will not use development section like Babel does by default when no NODE_ENV is set.

Observa: Si ha desactivado la transpilación de módulos ES6 con la opción { "modules": false }, debe asegurarse de activarlo en su entorno de prueba.

{
  "presets": [["env", {"modules": false}], "react"],
  "env": {
    "test": {
      "presets": [["env"], "react"]
    }
  }
}

Nota: babel-jest es instalado automáticamente cuando se instala Jest y convertirá automáticamente los archivos si su proyecto cuenta con una configuración babel. To avoid this behavior, you can explicitly reset the transform configuration option:

// package.json
{
  "jest": {
    "transform": {}
  }
}

Usando webpack

Jest puede ser usado en proyectos que usan webpackpara manejar bienes, estilos y compilación. webpack brinda algunos desafíos únicos sobre otras herramientas. Consulte la webpack guide para empezar.

Usando TypeScript

Para usar TypeScript en sus pruebas, puede usar ts-jest.
