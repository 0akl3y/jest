Testing React Native Apps

在 Facebook，我们使用 Jest 测试 React 应用。

阅读以下系列文章来深入了解如何使用Jest测试一个真实的React Native示例应用： 第一篇： Jest – Snapshot come into play 和 第二篇： Jest – Redux Snapshots for your Actions and Reducers.

安装

从react-native的0.38版本开始，使用react-native init初始化的项目默认已经包含了Jest测试框架。 以下配置应该已经被自动添加到您的 package. json 文件：

// package.json
  "scripts": {
    "test": "jest"
  },
  "jest": {
    "preset": "react-native"
  }

注意：如果你要升级一个使用了"preset":"jest-react-native" 的项目，应当从package.json中删除它，将preset换成react-native.

Simply run npm test to run tests with Jest.

Snapshot Test

下面来为一个入门的小型组件创建一个快照测试，它的内部有些View、Text组件和一些样式：

// Intro.js
import React, {Component} from 'react';
import {StyleSheet, Text, View} from 'react-native';

const styles = StyleSheet.create({
  container: {
    alignItems: 'center',
    backgroundColor: '#F5FCFF',
    flex: 1,
    justifyContent: 'center',
  },
  instructions: {
    color: '#333333',
    marginBottom: 5,
    textAlign: 'center',
  },
  welcome: {
    fontSize: 20,
    margin: 10,
    textAlign: 'center',
  },
});

export default class Intro extends Component {
  render() {
    return (
      <View style={styles.container}>
        <Text style={styles.welcome}>Welcome to React Native!</Text>
        <Text style={styles.instructions}>
          This is a React Native snapshot test.
        </Text>
      </View>
    );
  }
}

Now let's use React's test renderer and Jest's snapshot feature to interact with the component and capture the rendered output and create a snapshot file:

// __tests__/Intro-test.js
import 'react-native';
import React from 'react';
import Intro from '../Intro';

// Note: test renderer must be required after react-native.
import renderer from 'react-test-renderer';

test('renders correctly', () =>{
  const tree = renderer.create(<Intro />).toJSON();
  expect(tree).toMatchSnapshot();
});

当你运行npm test或者jest，将产生一个像下面的文件：

// __tests__/__snapshots__/Intro-test.js.snap
exports[`Intro renders correctly 1`] = `
<View
  style={
    Object {
      "alignItems": "center",
      "backgroundColor": "#F5FCFF",
      "flex": 1,
      "justifyContent": "center",
    }
  }>
  <Text
    style={
      Object {
        "fontSize": 20,
        "margin": 10,
        "textAlign": "center",
      }
    }>
    Welcome to React Native!
  </Text>
  <Text
    style={
      Object {
        "color": "#333333",
        "marginBottom": 5,
        "textAlign": "center",
      }
    }>
    This is a React Native snapshot test.
  </Text>
</View>
`;

The next time you run the tests, the rendered output will be compared to the previously created snapshot. The snapshot should be committed along code changes. When a snapshot test fails, you need to inspect whether it is an intended or unintended change. If the change is expected you can invoke Jest with jest -u to overwrite the existing snapshot.

The code for this example is available at examples/react-native.

Preset configuration

preset 定义的开发环境是我们根据在Facebook内部的使用经验生成的，并未考虑更为普适的场景。 所有配置选项都可以被改写, 和在没有使用预设时的自定义方式一致。

Environment

react-native 初始项目已经定义好了preset, 所以你的项目中package.json的jest.preset 字段值应为 react-native. preset指明了一个模仿出React Native应用运行环境的node环境。它无需加载任何DOM或浏览器接口，使得Jest的初始化时间大为缩短。

transformIgnorePatterns customization

transformignorepatterns 选项可用于创建babel转换的黑白名单。 但遗憾的是npm上很多react-native的包都不会在发布前编译源码。

jest-react-native默认只处理项目本身和react-native的代码。 如果你需要转换某些项目依赖，可以制定react-native项目以外的转换白名单：

"transformIgnorePatterns": [
  "node_modules/(?!(react-native|my-project|react-native-button)/)"
]

setupFiles

如果要为所有测试文件提供额外的配置, 则可以使用 setupfiles 配置选项 指定安装脚本。

moduleNameMapper

可以使用 modulenamemapper 将模块路径映射到其他模块。 默认情况下, 预设将所有图像映射到图像根模块, 但如果找不到模块, 此配置选项可以帮助：

"moduleNameMapper": {
  "my-module.js": "<rootDir>/path/to/my-module.js"
}

Tips

Mock native modules using jest.mock

构建在react-native项目中的jest预设已经含有一些模拟数据。 但是某些react-native模块或其他的第三方模块在原生代码渲染之后才能工作。 在这种情况下，Jest的手动模拟系统可以模拟出底层的实现。

比如你的项目依赖了一个名为react-native-video的原生视频组件，你应该会想要这样来模拟它的行为：

jest.mock('react-native-video', () => 'Video');

这么做你会得到一个<Video {...props} />组件，props被传入快照导出的所有属性。 See also caveats around Enzyme and React 16.

Sometimes you need to provide a more complex manual mock. 比如如果你想将原生组件的属性类型或静态数据传入模拟组件，你可以用过jest-react-native的帮助工具返回一个不同的React组件。

jest.mock('path/to/MyNativeComponent', () => {
  const mockComponent = require('react-native/jest/mockComponent');
  return mockComponent('path/to/MyNativeComponent');
});

Or if you'd like to create your own manual mock, you can do something like this:

jest.mock('Text', () => {
  const RealComponent = require.requireActual('Text');
  const React = require('React');
  class Text extends React.Component {
    render() {
      return React.createElement('Text', this.props, this.props.children);
    }
  }
  Text.propTypes = RealComponent.propTypes;
  return Text;
});

或者说你想想要模拟一个不是React组件的原生模块， The same technique can be applied. 我们建议检查本机模块的源码并在实际的设备上运行react-native应用程序时对该模块进行打日志, 然后依照实际模块建模进行手动模拟。

受够了针对同样的模块反复做数据模拟？推荐你将模拟数据保存成单独的文件，并将其加入setupFiles列表。

require react-native before the test renderer

目前为止，在引用test render之前，要先引用react-native：

import 'react-native';
// Require after react-native
import renderer from 'react-test-renderer';

@providesModule

如果您希望通过 npm 包使用 Facebook 的 @providesModule 模块系统, 则必须覆盖默认的 haste 选项, 并且必须将 npm 模块添加到 providesModuleNodeModules 中:

"haste": {
  "defaultPlatform": "ios",
  "platforms": ["android", "ios"],
  "providesModuleNodeModules": [
    "react",
    "react-native",
    "my-awesome-module",
    "my-text-component"
  ]
},

如果要测试不同的默认平台, 或者为其他平台构建, 则可以更新 defaultPlatform 和 platform 配置选项。
