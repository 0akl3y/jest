Getting Started

使用 npm 来安装 Jest︰

npm install --save-dev jest

或者通过yarn：

yarn add --dev jest

让我们先从写一个两个数相加的示例函数开始。首先，要创建一个 sum.js文件。

function sum(a, b) {
  return a + b;
}
module.exports = sum;

然后，创建一个名为 sum.test.js 的文件。这将包含我们的实际测试︰

const sum = require('./sum');

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});

将下面的配置部分添加到你的 package.json 里面：

{
  "scripts": {
    "test": "jest"
  }
}

最后，运行 npm test，Jest将会打印以下消息︰

PASS  ./sum.test.js
✓ adds 1 + 2 to equal 3 (5ms)

你刚刚成功地写了第一个Jest测试 ！

这个测试使用了 expect 和toBe 来测试两个值是否完全相同。 想要了解Jest能够测试的更多其他内容，请参考使用匹配器。

在命令行运行

你可以直接通过 CLI 运行 Jest (如果它已经在你的全局路径 PATH 中的话，比如通过 npm install -g jest 来安装) ，并指定各种有用的选项。

这里演示了如何在那些匹配 my-test 的文件上运行 Jest、使用config.json 作为配置文件、并在运行后显示一个本地操作系统通知。

jest my-test --notify --config=config.json

如果你愿意了解更多关于通过命令行运行 jest 的内容，请继续阅读 Jest CLI 选项 页面。

更多配置

使用 Babel

要使用 Babel，请安装 babel-jest 和 regenerator-runtime 两个包：

npm install --save-dev babel-jest babel-core regenerator-runtime

注意: 如果您使用的是 babel 7, 那么您需要用下面的命令安装 babel-jest:

npm install --save-dev babel-jest 'babel-core@^7.0.0-0' @babel/core regenerator-runtime

注意：如果您在使用 npm 3 或 4 ，或者 Yarn，您就不用再显式地安装 regenerator-runtime了。

别忘了在你的项目根文件夹下添加一个 .babelrc 配置文件。 举个例子，如果您在使用 ES6 和 React.js 并且配置了 babel-preset-es2015 和 babel-preset-react ：

{
  "presets": ["es2015", "react"]
}

现在你就完成了使用所有 ES6 特性和 React 特殊语法所需的配置了。

注意：如果你正在使用一个更复杂的 Babel 配置，并使用 Babel 的 env 选项，请记住 Jest 将自动把 NODE_ENV 设为 test。 它不会像 Babel 那样在 NODE_ENV 没有被设置时默认设为development 。

注意：如果你用 { "modules": false } 关闭了 ES2015 模块的转译，就必须确保在测试环境中将其开启。

{
  "presets": [["es2015", {"modules": false}], "react"],
  "env": {
    "test": {
      "presets": [["es2015"], "react"]
    }
  }
}

注意：当你安装 Jest 时，babel-jest 是会被自动安装的，并且如果你的项目下存在 Babel 配置文件，它将会自动对相关文件进行转译。 如果要避免这个行为，你可以显式的重置 transform 配置项：

// package.json
{
  "jest": {
    "transform": {}
  }
}

使用 webpack

Jest can be used in projects that use webpack to manage assets, styles, and compilation. webpack 与其他工具相比多了一些独特的挑战。 参考 webpack 指南 来开始起步。

使用 TypeScript

如果要在测试里用 TypeScript ，你可以用 ts-jest。
