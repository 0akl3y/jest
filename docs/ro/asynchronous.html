Testarea codului asincron

Se întâmplă frecvent în JavaScript să se execute cod asincron. Atunci când aveţi cod care se execută în mod asincron, Jest trebuie să ştie când codul pe care-l testează s-a completat de executat, înainte de a trece la un alt test. Jest are mai multe moduri pentru a rezolva acest lucru.

Callback-uri #

Cel mai frecvent tipar asincron este callback-ul.

De exemplu, să spunem că aveţi o funcţie fetchData(callback) care preia unele date şi apelează callback(data) la final. Doriţi să testaţi că valoarea returnată este şirul 'peanut butter'.

În mod implicit, testele Jest se finalizează odată ce se ajunge la sfârşitul execuției lor. Asta înseamnă că acest test nu va funcţiona așa cum ne-am aștepta:

// Don't do this!
test('the data is peanut butter', () => {
  function callback(data) {
    expect(data).toBe('peanut butter');
  }

  fetchData(callback);
});

Problema este că testul se va termina de îndată ce fetchData își termină execuția, înainte de a apela vreodată callback-ul.

Există o formă alternativă de test care rezolvă acest lucru. În loc de a pune testul într-o funcţie cu un argument gol, utilizaţi un singur argument numit done. Jest va aştepta până când funcția done este apelată înainte de terminarea testului.

test('the data is peanut butter', done => {
  function callback(data) {
    expect(data).toBe('peanut butter');
    done();
  }

  fetchData(callback);
});

În cazul în care done() nu este niciodată apelat, testul va eşua, ceea ce chiar doriți să se întâmple.

Promisiuni #

În cazul în care codul folosește promisiuni, există un mod mai simplu să tratați testele asincron. Doar returnați o promisiune din test, şi Jest va aştepta ca promisiunea să se rezolve. În cazul în care promisiunea este respinsă, testul va eşua automat.

De exemplu, să presupunem că fetchData, în loc să utilizeze un callback, returnează o promisiune care ar trebui să rezolve cu un şir 'peanut butter'. Am putut testa în felul următor:

test('the data is peanut butter', () => {
  expect.assertions(1);
  return fetchData().then(data => {
    expect(data).toBe('peanut butter');
  });
});

Asiguraţi-vă că returnați o promisiune - dacă omiteţi declaraţia return, testul se va termina înainte ca fetchData să se finalizeze.

Dacă vă aşteptaţi ca o promisiune să fie respinsă utilizați metoda .catch. Asiguraţi-vă că utilizați expect.assertions pentru a verifica că un anumit număr de aserțiuni sunt executate. Altfel o promisiune rezolvată nu ar eşua testul.

test('the fetch fails with an error', () => {
  expect.assertions(1);
  return fetchData().catch(e =>
    expect(e).toMatch('error')
  );
});

.resolves / .rejects #

disponibil în Jest 20.0.0+ #

Puteţi utiliza, de asemenea, validatorii .resolves în declaraţia expect, şi Jest va aştepta ca promisiunea să se rezolve. În cazul în care promisiunea este respinsă, testul va eşua automat.

test('the data is peanut butter', () => {
  expect.assertions(1);
  return expect(fetchData()).resolves.toBe('peanut butter');
});

Asiguraţi-vă că returnați aserțiunea - dacă omiteţi declaraţia return, testul se va termina înainte ca fetchData să se finalizeze.

Dacă vă aşteptaţi ca promisiunea să fie respinsă utilizați validatorul .rejects. Funcționează exact ca validatorul .resolves. În cazul în care promisiunea este îndeplinită, testul va eşua automat.

test('the fetch fails with an error', () => {
  expect.assertions(1);
  return expect(fetchData()).rejects.toMatch('error');
});

Async/Await #

Alternativ, puteţi utiliza async şi await în teste. Pentru a scrie un test asincron, folosiți cuvântul cheie async în faţa funcţiei din test. De exemplu, acelaşi scenariu fetchData poate fi testat cu:

test('the data is peanut butter', async () => {
  expect.assertions(1);
  const data = await fetchData();
  expect(data).toBe('peanut butter');
});

test('the fetch fails with an error', async () => {
  expect.assertions(1);
  try {
    await fetchData();
  } catch (e) {
    expect(e).toMatch('error');
  }
});

Desigur, puteţi combina async şi await cu .resolves sau .rejects (disponibil în Jest 20.0.0+).

test('the data is peanut butter', async () => {
  expect.assertions(1);
  await expect(fetchData()).resolves.toBe('peanut butter');
});

test('the fetch fails with an error', async () => {
  expect.assertions(1);
  await expect(fetchData()).rejects.toMatch('error');
});

În aceste cazuri, async şi await sunt doar variante simplificate ale codului care utilizează promisiuni.

Niciuna din aceste forme nu este în mod deosebit superioară celorlalte, şi le puteţi combina în diverse moduri. Totul depinde de stilul care face testele mai simple.
