Mock Functions

Funcţiile mock sau dubluri facilitează testarea legăturilor din cod prin înlocuirea implementării efective a unei funcţii, captarea apelurilor acelei funcţii (şi parametrii transmiși în aceste apeluri), captarea instanţele funcțiilor constructor atunci când se instanţiază cu new, şi permit configurarea valorilor returnate în teste.

There are two ways to mock functions: Either by creating a mock function to use in test code, or writing a manual mock to override a module dependency.

Utilizarea unei funcţii mock #

Să ne imaginăm că testăm implementarea unei funcţii forEach, care invocă un apel pentru fiecare element dintr-un array furnizat.

function forEach(items, callback) {
  for (let index = 0; index < items.length; index++) {
    callback(items[index]);
  }
}

Pentru a testa această funcţie, putem folosi o funcție mock şi a o inspecta pentru a se asigura că apelul este invocat așa cum ne aşteptăm.

const mockCallback = jest.fn();
forEach([0, 1], mockCallback);

// Funcția mock este apelată de 2 ori
expect(mockCallback.mock.calls.length).toBe(2);

// Primul argument al primului apel este 0
expect(mockCallback.mock.calls[0][0]).toBe(0);

// Primul argument al celui de al doilea apel este 1
expect(mockCallback.mock.calls[1][0]).toBe(1);

Proprietatea .mock #

Toate funcţiile mock au această proprietate specială .mock, în care se păstrează informațiile despre modul în care funcţia a fost apelată. Proprietatea .mock păstrează, de asemenea, și valoarea lui this pentru fiecare apel, astfel încât este posibilă inspecția lui, de asemenea:

const myMock = jest.fn();

const a = new myMock();
const b = {};
const bound = myMock.bind(b);
bound();

console.log(myMock.mock.instances);
// > [ <a>, <b> ]

Aceste proprietăți ale mock-ului sunt foarte utile pentru a verifica modul în care aceste funcţii au fost apelate, sau instanţiate:

// Funcția a fost apelată o singură dată
expect(someMockFunction.mock.calls.length).toBe(1);

// Primul argument al primului apel al funcției a fost 'primul arg'
expect(someMockFunction.mock.calls[0][0]).toBe('primul arg');

// Al doilea argument al primului apel al funcției a fost 'al doilea arg'
expect(someMockFunction.mock.calls[0][1]).toBe('al doilea arg');

// Funcția a fost apelată de exact 2 ori
expect(someMockFunction.mock.instances.length).toBe(2);

// Obiectul returnat de primul apel al funcției
// a avut o proprietate `name` a cărei valoare a fost 'test'
expect(someMockFunction.mock.instances[0].name).toEqual('test');

Returnarea valorilor mock #

Funcţiike mock pot fi folosite pentru a injecta valori de test în codul vostru în timpul unui test:

const myMock = jest.fn();
console.log(myMock());
// > undefined

myMock.mockReturnValueOnce(10)
 .mockReturnValueOnce('x')
 .mockReturnValue(true);

console.log(myMock(), myMock(), myMock(), myMock());
// > 10, 'x', true, true

Funcţiile mock sunt, de asemenea, foarte eficiente în codul care utilizează un stil funcțional continuation-passing. Codul scris în acest stil ajută la evitarea utilizării unor dubluri complicate pentru recrearea comportamentul componentei reale pe care o înlocuiesc, în favoarea injectării valorii direct în test chiar înainte ca acestea să fie utilizate.

const filterTestFn = jest.fn();

// Make the mock return `true` for the first call,
// and `false` for the second call
filterTestFn
  .mockReturnValueOnce(true)
  .mockReturnValueOnce(false);

const result = [11, 12].filter(filterTestFn);

console.log(result);
// > [11]
console.log(filterTestFn.mock.calls);
// > [ [11], [12] ]

Cele mai multe exemple reale implică de fapt obținerea legăturii funcției mock pe o componentă dependentă şi configurarea ei, dar tehnica este aceeaşi. În aceste cazuri, încercați să evitați tentaţia de a implementa orice logică în interiorul funcției care nu este testată direct.

Implementarea dublurilor mock #

Totuși, există cazuri în care este nevoie sa depășim simpla specificare a valorii returnate şi trebuie să înlocuim complet implementarea funcției mock. Acest lucru se poate realiza cu jest.fn sau metoda mockImplementationOnce pe funcţii mock.

const myMockFn = jest.fn(cb => cb(null, true));

myMockFn((err, val) => console.log(val));
// > true

myMockFn((err, val) => console.log(val));
// > true

Metoda mockImplementation este utilă atunci când aveţi nevoie să definiți implementarea implicită a unei funcţii mock, care este creată într-un alt modul:

// foo.js
module.exports = function() {
  // implementarea originală;
};

// test.js
jest.mock('../foo'); // acest lucru se întâmplă automat cu automocking
const foo = require('../foo');

// foo este o funcție mock
foo.mockImplementation(() => 42);
foo();
// > 42

Atunci când aveţi nevoie să reproduceți un comportament complex al unei funcții mock, astfel încât apelări diferite produc rezultate diferite, utilizaţi metoda mockImplementationOnce:

const myMockFn = jest.fn()
  .mockImplementationOnce(cb => cb(null, true))
  .mockImplementationOnce(cb => cb(null, false));

myMockFn((err, val) => console.log(val));
// > true

myMockFn((err, val) => console.log(val));
// > false

Atunci când funcţia mock rămâne fără implementări definite cu mockImplementationOnce, se va executa implementarea implicită setată cu jest.fn (în cazul în care este definită):

const myMockFn = jest.fn(() => 'implicit')
  .mockImplementationOnce(() => 'primul apel')
  .mockImplementationOnce(() => 'al doilea apel');

console.log(myMockFn(), myMockFn(), myMockFn(), myMockFn());
// > 'primul apel', 'al doilea apel', 'implicit', 'implicit'

Pentru cazurile în care avem metode care sunt de obicei înlănțuite (şi astfel fiind nevoie să returneze this), avem un API specific pentru a simplifica acest lucru sub forma funcţiei .mockReturnThis(), care, de asemenea, poate fi folosită pe orice pe orice mock:

const myObj = {
  myMethod: jest.fn().mockReturnThis(),
};

// is the same as

const otherObj = {
  myMethod: jest.fn(function() {
    return this;
  }),
};

Mock Names #

available in Jest 21.3.0+ #

You can optionally provide a name for your mock functions, which will be displayed instead of "jest.fn()" in test error output. Use this if you want to be able to quickly identify the mock function reporting an error in your test output.

const myMockFn = jest.fn()
  .mockReturnValue('default')
  .mockImplementation(scalar => 42 + scalar)
  .mockName('add42');

Validatori personalizați #

În cele din urmă, în scopul simplificării aserțiunilor pe funcţii mock, am adăugat unele funcţii personalizate:

// The mock function was called at least once
expect(mockFunc).toBeCalled();

// The mock function was called at least once with the specified args
expect(mockFunc).toBeCalledWith(arg1, arg2);

// The last call to the mock function was called with the specified args
expect(mockFunc).lastCalledWith(arg1, arg2);

// All calls and the name of the mock is written as a snapshot
expect(mockFunc).toMatchSnapshot();

Acești validatori sunt doar formule simplificate ale metodelor comune de a inspecta proprietatea .mock. Puteţi face întotdeauna aceste validări manual dacă preferați sau dacă aveţi nevoie să faceți ceva mai specific:

// The mock function was called at least once
expect(mockFunc.mock.calls.length).toBeGreaterThan(0);

// The mock function was called at least once with the specified args
expect(mockFunc.mock.calls).toContain([arg1, arg2]);

// The last call to the mock function was called with the specified args
expect(mockFunc.mock.calls[mockFunc.mock.calls.length - 1]).toEqual(
  [arg1, arg2]
);

// The first arg of the last call to the mock function was `42`
// (note that there is no sugar helper for this specific of an assertion)
expect(mockFunc.mock.calls[mockFunc.mock.calls.length - 1][0]).toBe(42);

// A snapshot will check that a mock was invoked the same number of times,
// in the same order, with the same arguments. It will also assert on the name. 
expect(mockFunc.mock.calls).toEqual([[arg1, arg2]]);
expect(mockFunc.mock.getMockName()).toBe('a mock name');
