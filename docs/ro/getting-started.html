Getting Started

Instalaţi Jest folosind npm:

npm install --save-dev jest

Sau prin yarn:

yarn add --dev jest

Să începem prin scrierea unui test pentru o funcţie ipotetică, pentru adaugarea a două numere. Mai întâi, creaţi un fişier sum.js:

function sum(a, b) {
  return a + b;
}
module.exports = sum;

Apoi, creaţi un fişier numit sum.test.js. Acesta va conţine testul nostru:

const sum = require('./sum');

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});

Adăugați secțiunea următoare în fișierul vostru package.json:

{
  "scripts": {
    "test": "jest"
  }
}

Apoi, rulați npm test iar Jest va afișa următorul mesaj:

PASS  ./sum.test.js
✓ adds 1 + 2 to equal 3 (5ms)

Tocmai ați scris cu succes primul test folosind Jest!

Testul acesta a folosit expect și toBe pentru a testa dacă două valori sunt identice. To learn about the other things that Jest can test, see Using Matchers.

Rularea din linia de comandă #

Puteți rula Jest direct din linia de comandă (dacă este disponibil global în PATH, de exemplu dacă ați folosit npm install -g jest) cu o varietate mare de opțiuni.

Iată cum să executaţi Jest pe fişierele my-test, folosind config.json ca fişier de configurare şi afişarea unei notificări native de sistem după rulare:

jest my-test --notify --config=config.json

If you'd like to learn more about running jest through the command line, take a look at the Jest CLI Options page.

Informații suplimentare #

Folosind Babel #

Pentru a utiliza Babel, instalaţi pachetele babel-jest şi regenerator-runtime:

npm install --save-dev babel-jest regenerator-runtime

Notă: Instalarea în mod explicit a regenerator-runtime nu este necesară în cazul în care utilizaţi npm 3 sau 4 sau Yarn

Nu uitaţi să adăugaţi fişierul .babelrc în directorul rădăcină din proiectul vostru. De exemplu, dacă utilizaţi ES6 şi React.js cu babel-preset-es2015 şi babel-preset-react:

{
  "presets": ["es2015", "react"]
}

Acum totul e pregătit pentru a folosi toate caracteristicile ES6 şi sintaxa specifică React.

Notă: Dacă utilizaţi o configurare Babel mai complicată, folosind optiunea Babel env, ţineţi minte că Jest va defini automat NODE_ENV ca test. Nu va folosi sectiunea development cum setează Babel în mod implicit, atunci când nu este setat explicit NODE_ENV.

Note: If you've turned off transpilation of ES2015 modules with the option { "modules": false }, you have to make sure to turn this on in your test environment.

{
  "presets": [["es2015", { "modules": false }], "react"],
  "env": {
    "test": {
      "presets": [["es2015"], "react"]
    }
  }
}

Notă: babel-jest se instalează automat când instalaţi Jest şi va transforma automat fişierele în cazul în care o configuraţie Babel există în proiectul vostru. Pentru a evita acest comportament, aveţi posibilitatea să reiniţializaţi explicit opţiunea de configurare transform:

// package.json
{
  "jest": {
    "transform": {}
  }
}

Folosind Webpack #

Jest poate fi utilizat în proiecte care folosesc webpack pentru gestionarea fișierelor statice, stiluri şi compilare. webpack oferă unele provocări unice față de alte instrumente. Refer to the webpack guide to get started.

Folosind TypeScript #

Pentru a utiliza TypeScript în teste utilizaţi ts-jest.
